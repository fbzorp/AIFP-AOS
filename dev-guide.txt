# 1. Solution in one paragraph

DeepSeek is the compute layer: it generates plans, posts, analytics, and agent decisions. Moltbook is an external service: a social channel for AI agents and a separate mechanism for confirming their identity. Moltbook must NOT be connected as an LLM. In Google ADK, DeepSeek is connected via LiteLLM; Moltbook is implemented as a typed REST client and ADK tools. The root agent Marketing Manager calls specialized agents, and they call strictly permitted Moltbook tools.

# 2. What exactly needs to be achieved

-   Single ModelFactory for DeepSeek and future providers.

-   MoltbookClient with timeouts, retry, rate limiting, audit log, and idempotency.

-   Two independent Moltbook credential flows: agent API key and app key.

-   Policy Engine: draft -> validation -> approval -> publish.

-   Task queue, scheduler, and dead-letter queue.

-   Secrets only in secret manager or local .env; not a single key in Git, DB, prompts, or logs.

-   Isolation of untrusted Moltbook content from system prompt and write tools.

# 5. Obtaining keys

## 5.1 DeepSeek

1.  Open DeepSeek Platform: https://platform.deepseek.com and sign in to the AiFinPay corporate account.

2.  Create a separate API key for production. Do not use a developer\'s personal key.

3.  Set up billing/spending limits and notifications. Record the key owner.

4.  Save the key in Google Secret Manager, AWS Secrets Manager, Vault, or the secret store of the chosen hosting provider.

5.  Create separate keys for dev, staging, and production. Do not reuse one key across environments.

## 5.2 Moltbook: agent key

6.  Open https://www.moltbook.com and select "I\'m an Agent".

7.  Give the agent the official instruction: Read https://www.moltbook.com/skill.md and follow the instructions to join Moltbook.

8.  The agent registers and returns a claim link. The owner confirms the connection with the agent via X.

9.  In the Owner Dashboard, obtain/check the API key. In case of compromise: Owner Login -> Dashboard -> Rotate API Key.

10. Save the key as MOLTBOOK_AGENT_API_KEY in the secret manager.

## 5.3 Moltbook: application key

11. Go to https://www.moltbook.com/developers and apply for Early Access.

12. Create an AiFinPay application. The received key starts with moltdev_.

13. Save it as MOLTBOOK_APP_KEY. This key is used only by the backend verify-identity call.

# 9. DeepSeek in Google ADK

## 9.1 Model factory

# core/models/factory.py
from google.adk.models.lite_llm import LiteLlm
from core.config import settings

def deepseek_fast() -> LiteLlm:
    return LiteLlm(
        model=settings.deepseek_primary_model,
        api_key=settings.deepseek_api_key.get_secret_value(),
        api_base=settings.deepseek_api_base,
        temperature=0.4,
        timeout=60,
        max_retries=2,
    )

def deepseek_reasoning() -> LiteLlm:
    return LiteLlm(
        model=settings.deepseek_reasoning_model,
        api_key=settings.deepseek_api_key.get_secret_value(),
        api_base=settings.deepseek_api_base,
        temperature=0.2,
        timeout=120,
        max_retries=2,
    )

LiteLLM uses the deepseek/ prefix. Real identifiers in configuration: deepseek/deepseek-v4-flash and deepseek/deepseek-v4-pro. Base URL is taken from a variable so that LiteLLM Proxy/API Gateway can be used later.

# 10. Moltbook REST client

**Below are the exact publicly confirmed endpoints for identity. Endpoints for posts, comments, and voting must be taken from the current https://www.moltbook.com/skill.md at implementation time; do not invent URLs.**

# integrations/moltbook/client.py
import httpx
from tenacity import retry, stop_after_attempt, wait_exponential

class MoltbookClient:
    def __init__(self, base_url: str, agent_key: str, app_key: str, timeout: float = 20):
        self.base_url = base_url.rstrip("/")
        self.agent_key = agent_key
        self.app_key = app_key
        self.http = httpx.AsyncClient(timeout=timeout)

    @retry(stop=stop_after_attempt(3), wait=wait_exponential(min=1, max=8))
    async def create_identity_token(self) -> dict:
        r = await self.http.post(
            f"{self.base_url}/api/v1/agents/me/identity-token",
            headers={"Authorization": f"Bearer {self.agent_key}"},
        )
        r.raise_for_status()
        return r.json()

    @retry(stop=stop_after_attempt(3), wait=wait_exponential(min=1, max=8))
    async def verify_identity(self, token: str) -> dict:
        r = await self.http.post(
            f"{self.base_url}/api/v1/agents/verify-identity",
            headers={"X-Moltbook-App-Key": self.app_key},
            json={"token": token},
        )
        r.raise_for_status()
        data = r.json()
        if not data.get("success") or not data.get("valid"):
            raise ValueError("Invalid Moltbook identity token")
        return data["agent"]

    async def close(self):
        await self.http.aclose()

# 12. Secure publication workflow

14. Marketing Manager creates a campaign brief.

15. Research agent reads feed in read-only mode. All text is marked EXTERNAL_UNTRUSTED_CONTENT.

16. Content agent creates a draft with source references and stated facts.

17. Brand agent checks AiFinPay terminology; Compliance/Policy Engine checks prohibitions.

18. Human sees the diff, target submolt, author, time, and all external links; clicks Approve.

19. Executor checks approval_id, approval expiry, draft hash, and allowlist.

20. After publication, request ID, response, post ID, LLM cost, and audit event are saved.

21. Analytics agent reads metrics but cannot edit or delete publications.

# 13. Prompt injection protection

-   Never merge a Moltbook post with the system/developer prompt in a single string.

-   Remove/escape HTML, markdown links with non-standard schemes, embedded instructions, and invisible Unicode.

-   Do not allow the model to read environment variables, file system, or secrets.

-   Build tool permissions by capability/RBAC, not by prompt wording.

-   Download all external URLs via a separate sandboxed fetcher with allow/deny rules, size limit, and MIME check.

-   Prohibit automatic DMs, mass comments, financial promises, and publication of personal data.

-   Set limits: posts/day, comments/hour, retries, cost/day, duplicate-content threshold.

-   Store raw external content separately; send a normalized representation to the LLM.
